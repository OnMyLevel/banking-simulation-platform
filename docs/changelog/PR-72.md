# PR #72 - Angular Authentication interceptor

## Files changed

- `frontend/angular-app/src/app/auth-token.service.ts`
- `frontend/angular-app/src/app/auth-token.service.spec.ts`
- `frontend/angular-app/src/app/auth.interceptor.ts`
- `frontend/angular-app/src/app/auth.interceptor.spec.ts`
- `frontend/angular-app/src/main.ts`
- `frontend/angular-app/README.md`

## Changes

- Adds an Angular token service for centralized bearer token access.
- Adds an Angular HTTP authentication interceptor.
- Adds `Authorization: Bearer <token>` to outgoing HTTP requests when a token is available.
- Preserves existing `Authorization` headers.
- Registers the authentication interceptor after the Correlation ID interceptor.
- Adds unit tests for token storage, token clearing and authentication header behavior.
- Adds integration-style interceptor coverage with the Correlation ID interceptor.
- Updates the Angular README.

## Behavior

```text
- no token: request is sent without Authorization header
- token available: request receives Authorization: Bearer <token>
- existing Authorization header: header is preserved
- Correlation ID and Authentication interceptors can run together
```

## Impact

- No backend runtime behavior change.
- Prepares authenticated Gateway requests from Angular.
- Prepares future refresh-token and 401 retry handling.
