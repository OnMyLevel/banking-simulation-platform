# PR #66 - Angular Gateway service

## Files changed

- `frontend/angular-app/src/app/gateway-api.service.ts`
- `frontend/angular-app/src/app/gateway-api.service.spec.ts`
- `frontend/angular-app/README.md`

## Changes

- Adds an Angular Gateway API service for advisor dashboard data.
- Converts successful Gateway responses into ready dashboard state.
- Converts empty responses into empty dashboard state.
- Converts Gateway errors into UI-ready mapped error state.
- Reuses the shared Angular error mapper added in PR #65.
- Adds service tests for ready, empty and error behavior.
- Updates the Angular README.

## Behavior

```text
- 200 with dashboard items returns ready state
- 204 returns empty state
- 200 with no dashboard items returns empty state
- non-2xx response returns mapped error state
- retry-after and correlation id are preserved when available
```

## Impact

- No backend runtime behavior change.
- No component visual behavior change.
- Prepares Angular for real Gateway-backed dashboard loading.
