# PR #73 - Angular Gateway HTTP resilience

## Files changed

- `frontend/angular-app/src/app/gateway-api.service.ts`
- `frontend/angular-app/src/app/gateway-api.service.spec.ts`
- `frontend/angular-app/README.md`

## Changes

- Adds timeout handling to Gateway dashboard HTTP calls.
- Adds retry handling for temporary Gateway failures.
- Retries network errors and temporary Gateway/server errors only.
- Avoids retrying client and business errors.
- Keeps final error mapping through the shared Angular Gateway error mapper.
- Adds unit tests for retryable and non-retryable responses.
- Updates the Angular README.

## Behavior

```text
- Gateway request times out after 5000 ms
- temporary failures are retried 2 times
- retryable failures: network errors, 502, 503, 504
- non-retryable failures: 400, 401, 403, 404, 429 and other client/business errors
- exhausted retry budget returns mapped error state
```

## Impact

- No backend runtime behavior change.
- Improves frontend resilience for temporary Gateway failures.
- Prepares future production-grade HTTP behavior around metrics and logging.
