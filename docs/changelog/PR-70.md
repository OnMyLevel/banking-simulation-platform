# PR #70 - Angular HttpClient Gateway service

## Files changed

- `frontend/angular-app/src/app/gateway-api.service.ts`
- `frontend/angular-app/src/app/gateway-api.service.spec.ts`
- `frontend/angular-app/src/app/app.component.ts`
- `frontend/angular-app/src/app/app.component.spec.ts`
- `frontend/angular-app/src/main.ts`
- `frontend/angular-app/README.md`

## Changes

- Migrates `GatewayApiService` from browser `fetch` to Angular `HttpClient`.
- Exposes advisor dashboard loading as an RxJS `Observable`.
- Updates `AppComponent` to consume the Gateway observable.
- Registers Angular `provideHttpClient()` at bootstrap.
- Replaces fetch-based service tests with Angular HTTP testing utilities.
- Updates app tests for observable Gateway state.
- Updates the Angular README.

## Behavior

```text
- Gateway service performs GET /api/advisor/dashboard through HttpClient
- 200 with items returns ready state
- 204 returns empty state
- 200 with no items returns empty state
- non-2xx response returns mapped error state
- retry-after and correlation id are preserved when available
```

## Impact

- No backend runtime behavior change.
- Aligns Angular data access with Angular platform conventions.
- Prepares future interceptors for auth, correlation id, logging, retry and timeout.
