# PR #67 - Angular dashboard Gateway integration

## Files changed

- `frontend/angular-app/src/app/app.component.ts`
- `frontend/angular-app/src/app/app.component.spec.ts`
- `frontend/angular-app/README.md`

## Changes

- Connects the Angular app root to `GatewayApiService`.
- Adds loading state while Gateway data is requested.
- Passes ready Gateway dashboard data into `AdvisorDashboardComponent`.
- Passes empty Gateway state into the dashboard component.
- Passes mapped Gateway error state into the dashboard component.
- Adds app-level tests for loading, ready, empty and error states.
- Updates the Angular README.

## Behavior

```text
- app starts in loading state
- ready Gateway result renders dashboard title and items
- empty Gateway result renders empty dashboard state
- error Gateway result renders mapped error title, message and reference
```

## Impact

- No backend runtime behavior change.
- Angular dashboard is now connected to the Gateway service layer.
- Prepares the app for real Gateway-backed advisor data in local/dev environments.
