# PR #71 - Angular Correlation ID interceptor

## Files changed

- `frontend/angular-app/src/app/correlation-id.interceptor.ts`
- `frontend/angular-app/src/app/correlation-id.interceptor.spec.ts`
- `frontend/angular-app/src/main.ts`
- `frontend/angular-app/README.md`

## Changes

- Adds an Angular HTTP interceptor for correlation IDs.
- Adds `X-Correlation-Id` to outgoing HTTP requests when absent.
- Preserves existing `X-Correlation-Id` headers when already provided.
- Registers the interceptor with Angular `provideHttpClient`.
- Adds unit tests for generated and preserved correlation IDs.
- Updates the Angular README.

## Behavior

```text
- outgoing HTTP requests receive X-Correlation-Id
- existing correlation IDs are not overwritten
- generated IDs use crypto.randomUUID when available
- fallback ID generation is available for environments without randomUUID
```

## Impact

- No backend runtime behavior change.
- Improves frontend/backend request traceability.
- Prepares future observability, logging and distributed tracing work.
